# 💰 DLOB Daily Attendance Fee System - Simplified Implementation

## 📋 Overview

**CHANGE:** Attendance fee structure modified from **per-match** to **per-day** payment model.

### Previous System (Per Match) ❌
- Member without membership plays 5 matches on Saturday
- Charged Rp 18,000 × 5 = **Rp 90,000** (attendance fees)
- Plus shuttlecock fees

### New System (Per Day) ✅  
- Member without membership plays 5 matches on Saturday
- Charged Rp 18,000 × **1** = **Rp 18,000** (attendance fee)
- Plus shuttlecock fees
- **Pay ONCE per Saturday**, regardless of number of matches

---

## 🗄️ Database Changes

### NO NEW TABLE NEEDED! ✅

We use the existing `match_members` and `matches` tables with just **3 new columns**:

### Updated Table: `matches`

```sql
ALTER TABLE matches
  ADD COLUMN match_number INTEGER;
  
ALTER TABLE matches
  ADD COLUMN match_code TEXT;

-- Trigger automatically generates match_code from match_date + match_number
CREATE TRIGGER trigger_set_match_code
  BEFORE INSERT OR UPDATE ON matches
  FOR EACH ROW EXECUTE FUNCTION set_match_code();
```

**What these columns mean:**
- `match_number`: Sequential number per day (#1, #2, #3... #15), resets each Saturday
- `match_code`: Auto-generated unique identifier (e.g., `20260222-05`), **use this for display!**
- Auto-set via trigger using `get_next_match_number()` function

**Example:**
- Saturday Feb 22, 2026: `20260222-01`, `20260222-02`, `20260222-03`... `20260222-15`
- Saturday Feb 29, 2026: `20260229-01`, `20260229-02`... `20260229-12` (unique codes!)

### Updated Table: `match_members`

```sql
ALTER TABLE match_members
  ADD COLUMN attendance_paid_this_entry BOOLEAN DEFAULT false;
```

**What this column means:**
- `TRUE` = This match entry CHARGED the attendance fee (first match of the day)
- `FALSE` = This match entry did NOT charge attendance (member already paid today)

## Match Identification System

Each match gets a **unique match_code** in format `YYYYMMDD-NN` (e.g., `20260222-05`).

### Why match_code?
- ✅ Unique across all time (no ambiguity between weeks)
- ✅ Human-readable and sortable
- ✅ Auto-generated by database
- ❌ Plain match numbers (#5) repeat every Saturday - confusing!

**See detailed guide:** [MATCH-CODE-SYSTEM.md](MATCH-CODE-SYSTEM.md)

### How it works:
- **Saturday Feb 22, 2026:**
  - First match → `20260222-01` (Match #1)
  - Second match → `20260222-02` (Match #2)
  - Fifth match → `20260222-05` (Match #5)
  - 15th match → `20260222-15` (Match #15)

- **Saturday Feb 29, 2026:**
  - First match → `20260229-01` (Match #1, different week!)
  - Second match → `20260229-02` (Match #2)
  - ...

### Benefits:
- Easy to identify matches ("Check out Match #7 results!")
- Clear ordering for match history
- Simple to count total matches per day
- Helps with scheduling and court assignments

### Query match numbers:
```sql
-- Get all match numbers for today
SELECT match_number, match_date, shuttlecock_count
FROM matches
WHERE DATE(match_date) = CURRENT_DATE
ORDER BY match_number;

-- Find a specific match by number
SELECT *
FROM matches
WHERE DATE(match_date) = '2026-02-22'
  AND match_number = 5;

-- Count total matches for a day
SELECT 
  DATE(match_date) as play_date,
  MAX(match_number) as total_matches
FROM matches
WHERE DATE(match_date) = '2026-02-22'
GROUP BY DATE(match_date);
```

### Get daily attendance summary:
```sql
SELECT 
  mm.member_name,
  COUNT(*) as total_matches,
  SUM(CASE WHEN mm.attendance_paid_this_entry THEN 1 ELSE 0 END) as times_charged,
  SUM(mm.attendance_fee) as total_attendance_paid,
  bool_or(mm.has_membership) as has_membership
FROM match_members mm
JOIN matches m ON mm.match_id = m.id
WHERE DATE(m.match_date) = '2026-02-22'
GROUP BY mm.member_name
ORDER BY mm.member_name;
```

**Expected output:**
| member_name | total_matches | times_charged | total_attendance_paid | has_membership |

| Adnan       | 3             | 0             | 0                     | TRUE           |
| John        | 2             | 1             | 18000                 | FALSE          |

### Mark attendance as paid:
```sql
UPDATE match_members
SET payment_status = 'paid'
WHERE member_name = 'Peno'
  AND attendance_paid_this_entry = true
  AND match_id IN (
    SELECT m.id FROM matches m 
    WHERE DATE(m.match_date) = '2026-02-22'
  );
```

## 🔒 Data Integrity

### Constraints in match_members:
- `member_name` TEXT NOT NULL
- `match_id` UUID REFERENCES matches(id)
- `attendance_paid_this_entry` BOOLEAN DEFAULT false

### Query to verify no duplicates:
```sql
-- Each member should have max 1 attendance payment per day
SELECT 
  mm.member_name,
  DATE(m.match_date) as match_date,
  COUNT(*) FILTER (WHERE mm.attendance_paid_this_entry = true) as payments
FROM match_members mm
JOIN matches m ON mm.match_id = m.id
GROUP BY mm.member_name, DATE(m.match_date)
HAVING COUNT(*) FILTER (WHERE mm.attendance_paid_this_entry = true) > 1;

-- Should return 0 rows (no duplicates)
```

## 🔒 Security & Constraints

### Unique Constraint Protection:
```sql
CONSTRAINT unique_daily_attendance UNIQUE (member_name, attendance_date)
```

**What this prevents:**
- Duplicate payments for same member on same day
- Race conditions when creating multiple matches simultaneously
- Data inconsistency

**How it works:**
- First INSERT succeeds
- Subsequent INSERTs for same (member, date) fail
- API handles error gracefully and uses existing payment

### RLS Policies:
- **SELECT:** Anyone can view (needed for lookups)
- **INSERT:** Anyone can create (system automation)
- **UPDATE:** Anyone can update (payment status changes)
- **DELETE:** Only admins
